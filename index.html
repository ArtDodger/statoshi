Statoshi

Bitcoin Core StatsD integration

Download this project as a .zip file Download this project as a tar.gz file

Installation and Dependencies

Install node.js http://nodejs.org/download/

Download / clone the statsd project from https://github.com/etsy/statsd

Create a config file copied from exampleConfig.js

Start the Daemon: node stats.js /path/to/your/config.js

Download and install Apache HTTPD Server from https://httpd.apache.org/download.cgi

If you’re running Linux, it’s probably easier to install via your package manager.

Now you’ll need to install software to made use of the stats collected by statsd. I recommend Graphite: http://graphite.wikidot.com/installation

Download and build the Bitcoin Statoshi fork:

Now you need to make sure you have the dependencies required to build Bitcoin Core.

Linux build notes: are listed here

OSX build notes: are listed here

Windows build notes: are listed here

Once you think you have all of the dependencies installed, build the project.

jameson@lopp:~/$ cd /path/to/statoshi/
jameson@lopp:~/workspace/bitcoin/$ ./autogen.sh
jameson@lopp:~/workspace/bitcoin/$ ./configure
jameson@lopp:~/workspace/bitcoin/$ make

It may take 20+ minutes to build, depending upon your computer’s specs. If you’re missing a dependency, you should receive an error noting what is missing.

Note that the Statoshi project assumes that you are running StatsD on the same machine as the Bitcoin Node. If you wish to run StatsD on a different machine or non-default port, you’ll need to edit the default configuration on line 16 of /statoshi/src/statsd_client.h before building the project.

Start your Statoshi node (either the GUI or the command line daemon)

/path/to/bitcoin-metrics/src/qt/bitcoin-qt &
/path/to/bitcoin-metrics/src/bitcoind &

Now you can check out the stats as they roll in to Graphite; you should see graphs begin to be populated within 30 seconds!
